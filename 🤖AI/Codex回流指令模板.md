# Codex 回流指令模板（自生长版）

把下面这段发给 **ChatGPT 里的 Codex**，让它读龙虾简报 + Inbox，加工成「每日观点 + 概念卡片」，写回 vault。

## 指令原文（复制给 Codex）

> 我的 Obsidian 笔记在 `/Users/Apple/Documents/file/LzgFile`。
>
> 请执行每日知识库加工任务：
>
> ### A. 消化来源
> 1. 读取 `💰金融/自动/` 下今天及最近的龙虾简报（金融新闻 / A股晨报 / A股复盘）。
> 2. 读取 `Inbox/` 下的碎片（若有）：判断每条该**归档到 `💰金融/01_Sources/` 或 `🤖AI/01_Sources/`**、提炼成知识卡片、或直接丢弃（无价值）。
>
> ### B. 产出每日观点（保留旧格式）
> 3. 综合提炼一份「每日观点」存到 `💰金融/Codex产出/Codex观点-YYYY-MM-DD.md`，含三段：
>    - **重点观点**：3-5 条宏观/市场判断
>    - **持仓关注**：板块/个股观察项（不荐股）
>    - **风险提示**：当日主要风险点
>
> ### C. 产出原子概念卡片（新增）
> 4. 从简报中抽 2-5 个值得长期沉淀的**概念**，每个写成一页 `💰金融/02_Knowledge/概念名.md`：
>    - frontmatter：`tags: [金融/知识卡片]`
>    - 内容：概念定义 + 当前市场含义 + 五维评估（相关性/新鲜度/价值度/可输出性/关联性，各 1-5 分）+ `[[双向链接]]` 到相关概念与 `[[💰金融/03_Topics]]`
>    - 若同名卡片已存在，**更新**而非新建（合并新信息）
>
> ### D. 护栏（必须）
> 5. 保留 `---` frontmatter、`[[双链]]`、`#标签` 语法不变，只写要求内容。
> 6. 绝不删除他人笔记、绝不改 `🏠 Home.md` / `MOC-*.md` 的结构。
> 7. 完成后汇报：写了哪些文件、更新了哪些卡片。

## 使用说明
- 龙虾 17:30 复盘简报同步后跑一次即可
- Codex 直接文件系统读写 vault，零配置
- 写完后 Obsidian Git 自动备份，龙虾下次 pull 读回（优先读 `💰金融/02_Knowledge/`）
