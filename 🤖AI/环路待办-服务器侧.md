# 环路待办：服务器侧（需 ssh lobster 配合）

以下两步需要在腾讯云龙虾服务器执行。当前 Mac 代理（Clash Verge TUN 模式）拦截 SSH 出站，需用户**临时关 TUN** 或用**网页终端**代跑后，我可一键执行。

## 待办 1：龙虾定时 pull 读回 Codex 产出

目标：龙虾生成推送前，先 `git pull --rebase`，使 `💰金融/Codex产出/` 的 Codex 观点能被读取，用于微信/飞书推送。

在 `~/sync_to_vault.sh` 或独立 cron 中追加：

```bash
# 龙虾读回 Codex 产出（在生成推送前执行）
cd ~/lzg-vault && git pull --rebase origin main
# 读取 💰金融/Codex产出/Codex观点-$(date +%F).md 作为推送素材
```

## 待办 2：极空间 NAS 第 3 备份

在极空间设备（或 Docker 容器）部署 git 镜像，作为 3-2-1 备份的第 3 副本（只读、不冲突）。

1. 生成独立**只读** deploy key（不碰 Mac 的 github_key / 龙虾的写 key）
2. 极空间定时 `git pull` `lzg-vault` 到本地目录，每天一次
3. 公钥加到 GitHub deploy key（**不勾** Allow write access）

生成命令（极空间或服务器执行）：

```bash
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_lzgvault_ro -N "" -C "zspace-lzgvault-ro"
cat ~/.ssh/id_ed25519_lzgvault_ro.pub
```
